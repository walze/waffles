## mediaQuery

Based on the **breakpoints** defined in [design tokens](/foundation/design-tokens) return media queries for various device viewport sizes: `aboveSmall` (tablets), `aboveMedium` (small desktops), and `aboveLarge` (wide screen).

Following mobile-first approach, when using multiple media queries make sure to write them in correct order: put the _mobile_ styles first (these are the ones without any media query specified), `aboveSmall` before `aboveMedium`, and `aboveMedium` before `aboveLarge`.

## readableHexColor

For the best contrast and readibility, depending whether provided **HEX** color is dark or light return `tokens.colors.white` or `tokens.colors.navy` from [design tokens](/foundation/design-tokens). It is also possible to specify custom colors.

## hexColorLuminance

Calculates **HEX** color luminance. Useful for various accessibility color checks.

## isHexColor

Checks whether color is provided in **HEX** notation or not. Examples of **HEX** colors: `#05192D`, `#22E`. Examples of colors in _different_ notations: `rgb(245, 16, 0)`, `rgba(16, 8, 36, 0.4)`, `hsl(100 100% 50%)`.

## deepChildrenMap

In a similar way to [React.Children.map](https://reactjs.org/docs/react-api.html#reactchildren), creates a new children array, but calls a provided function recursively on every child (and its children).
